Position

  1. Relative
    2. Positioned relative to their normal position
    in the document flow.
    Can move with "top, left, right.."
  2. Static
    3. Positioned according to the normal flow
    of the document.
    Cannot use with "top, left, right.."
  3. Absolute
    4. Realtive to their father (not static)
    removed from the normal flow,
    overlap other elemnent.
    Can move with "top, left, right.."
  4. fixed
    5. Relative to the viewport,
    Fixed in scroll.
    Commonly used for headers, footers, or elements
    that should always be visible.
  5. Sticky
    6. Based on the user's scroll position.
    Behave like relative until they reach
    a specified scroll position, then they become fixed.
    Useful for headers or navigation

Display

  1. Block
    2. Start on a new line and stretch
    the full width of their containing element
  2. Inline
    3. Flow within the text and only take
    up as much width as necessary.
  3. Inline-block
    4. Similar to 'inline' but behave like 'block',
    Useful for creating inline elements
    with block-level styling
  4. None
    5. Hidden, useful for hiding elements
    dynamically with JavaScript or CSS
    like this hover
  5. Flex
    6. flexible containers for their child elements.
    for easy creation of flexible and responsive
  6. Grid
    7. layout with rows and columns
  7. Table
    8. Like sizes on left
